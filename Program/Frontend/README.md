# Investigative Protocol Protection Client

Desktop-клиент на Avalonia и .NET 10 для защиты протоколов любых следственных действий: допроса, обыска, выемки, осмотра и других документов.

## Реализовано

- вход через Keycloak Authorization Code + PKCE;
- список и скачивание документов из backend API;
- локальное полное и частичное шифрование AES-256-GCM через `System.Security.Cryptography`;
- импорт TXT, MD, DOCX, ODT и самодостаточных ENC-контейнеров;
- экспорт расшифрованного текста в DOCX и ODT с сохранением доступного форматирования;
- контроль целостности, журнал действий, поиск и фильтрация журнала;
- автоматический выход после 15 минут бездействия;
- отменяемая асинхронная загрузка и обработка документов без блокировки интерфейса.

Примеры документов: протокол допроса свидетеля, протокол обыска помещения, протокол выемки предметов, протокол осмотра места происшествия.

## Серверный контракт

Режимы backend: `Full = 1`, `Part = 2`, `None = 3`. Размер файла проверяется до отправки; предел nginx составляет 20 МБ, ответ `413 Payload Too Large` должен показываться пользователю отдельно.

Целевой сценарий `Full`: создать 32-байтовый секрет, зашифровать документ на клиенте, загрузить `.enc`; при ошибке загрузки удалить созданный секрет.

Целевой сценарий `Part`: создать секрет, заменить выбранные участки маркерами, загрузить документ с маркерами, затем отправить шифротексты через Fragment API.

Целевой сценарий `None`: загрузить исходный файл без шифрования.

При открытии защищённого документа клиент получает метаданные через `GET /v1/documents/{id}`, читает `SecretId`, получает ключ и выполняет расшифрование локально. Клиент никогда не отправляет открытый текст защищаемого фрагмента на сервер.

## Запуск и проверка

```powershell
dotnet run --project .\Interrogation.Client\Interrogation.Client.csproj
dotnet build .\Interrogation.Frontend.slnx -c Release
dotnet run --project .\Interrogation.Client.SmokeTests\Interrogation.Client.SmokeTests.csproj
```

После backend-коммита `fc9351f` локальное окружение требует однократного пересоздания хранилищ: `docker compose down -v`, затем `docker compose up`. Команда удаляет локальные данные Docker.
